# 상속(Inheritance)

```java
public class Parent {}

public class Child extends Parent {}

```

## `extends` 키워드

- **"확장하다"** 라는 뜻에 집중하면 더 이해하기 쉽다.
- 기존 기능에 더해 자식 클래스에서 기능을 확장한다는 의미

## 상속 규칙

1. 단일 상속만 가능 -> 인터페이스 사용

2. 생성자와 초기화 블록은 상속되지 X (상속되는 것: 필드, 메서드)

```java
class Parent {
  int a = 10;   // 멤버 변수(필드) - 상속 O

  { /* 인스턴스 초기화 블록 - 상속 X */ }

  public Parent() {}  // 생성자 - 상속 X
}
```

- 생성자가 상속된다면? 생성자는 class명과 같아야하는데 문법적으로 맞지 않음
- 초기화 블록은 객체 생성 할 때만 실행되는 초기화 절차일 뿐

3. private 멤버는 상속은 되지만 접근 X

- getter/setter 통해서 접근

## `super` 키워드
부모 클래스의 멤버에 접근 할 떄 사용

### 1. `super`

- `super` : 부모 인스턴스에 대한 **참조** 변수
- `thils` : 인스턴스 자신에 대한 참조 변수

### 2. `super( )`

자식 클래스 생성자에서 부모 클래스의 **생성자 호출문**

- 생성자 호출문 : 생성자 안에서 다른 생성자를 실행하고 객체 초기화하는 문장
  - 객체 초기화 순서를 보장하기 위해 이 중 하나만, 생성자 첫줄에 위치
  - `super()` : 부모 클래스의 생성자 호출
  - `this()` : 같은 클래스의 다른 생성자 호출

## 메서드 오버라이딩 (Overriding)

**오버로딩이랑 차이점은?**

- **오버라이딩** : 부모 클래스의 메서드를 자식 클레스에서 재정의
- 오버로딩 : 같은 이름의 메서드를 다른 매개변수로 여러 개 정의


**오버라이딩 규칙**
- static 메서드의 경우 재정의 불가 (별개의 메서드가 됨)

### `@Override` 어노테이션
어노테이션은 컨파일러에게 특정 동작을 알리는 표식

## `final` 키워드
final 메서드, 클래스는 오버라이딩 및 상속 할 수 X

## Object 클래스
모든 클래스는 암묵적으로 `Object` 클래스를 상속 받음

- toString() : 객체 문자열 반환
- equals() : 참조 비교
