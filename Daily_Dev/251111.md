# Daily Dev Log

## 느낀 점 & 회고 (Thoughts & Reflections)

#### 1. 동적 바인딩 VS 정적 바인딩

```java
class Parent {
    String field = "Parent field";
}

class Child extends Parent {
    String field = "Child field";
}

public class FieldBinding {
    public static void main(String[] args) {
        Parent p = new Child();
        System.out.println(p.field);  // "Parent field" (필드는 정적 바인딩)

        Child c = (Child) p;
        System.out.println(c.field);  // "Child field"
    }
}
```

- **어려웠던 점 (Difficulties):**
필드는 왜 정적 바인딩인지 이해가 되지 않았다. 
- **깨달은 점 (Realization):**
애초에 '오버라이딩'이라는 개념은 메서드에 쓰려고 만든 개념이다. 필드는 '객체의 상태'기 때문의 참조 변수의 타입으로 결정되는 것이 당연히 자연스러운 상황이다.

- **정리**
  - 필드 = 상자 안 물건 -> 어떤 상자(참조 타입)인지가 중요
  - 메서드 = 명령어 실행 -> 어디서 명령한건지가(실제 객체) 중요
